# Audience And Expository Contract

## Readers

The primary human readers are advanced PhD students and researchers in
theoretical physics, mathematical physics, or adjacent areas. They are assumed
to be mathematically competent, but they are not assumed to know operator
algebras, microlocal analysis, higher category theory, TQFT, derived geometry,
or deep functional analysis before the book introduces those tools.

AI agents are also intended readers. The manuscript and planning layer should
allow agents to audit source provenance, notation, assumptions, figures,
claims, dependencies, and unresolved debt.

## Reader Promise

The book should make hard ideas navigable without lowering the standard. A
reader should be able to answer:

- What framework is being used here?
- Which objects are primitive and which are constructed?
- What is the type of each symbol?
- What assumptions are required for this claim?
- Is this statement exact, regulated, perturbative, formal, heuristic,
  conjectural, or open?
- Where was this notation introduced?
- What changes when the framework changes?

## Definition Contract

Every reader-facing definition should specify:

- ambient setting;
- input data and their types;
- maps, domains, codomains, and support conditions;
- algebraic, analytic, covariance, locality, positivity, and spectral
  assumptions when relevant;
- whether the object is primitive for the current framework or constructed from
  earlier data;
- examples that verify the definition rather than merely decorate it.

The book should never use a "simplified" definition as a substitute for the
definition actually needed. If the complete mathematical object is not
available, the text should give a clearly labelled working framework and state
what remains unresolved.

## Cognitive Burden

The manuscript reduces cognitive burden by organization, not by weakening the
content. It should:

- use stable notation;
- avoid hidden convention changes;
- stage prerequisites before they become load-bearing;
- separate exact statements from interpretation;
- place optional technical depth in interludes or appendices;
- provide local roadmaps before long derivations;
- keep figures precise and labeled;
- use examples that test assumptions.

## Voice

The voice should be direct, rigorous, and constructive. Criticism of standard
textbook presentations is allowed when it prevents a concrete misreading, but
the main prose should be organized around correct objects and derivations.

The manuscript should not say in the main flow what the subject is "not" when a
positive definition or domain statement is available. Remarks and footnotes may
record common traps after the correct formulation has been given.
